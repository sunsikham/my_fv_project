# Multi-Feature Theory

## Main Claim

The current strongest interpretation is:

> Context does not simply replace A with one B-only or D-only axis.
> Instead, a substantial part of the original A-structure remains, while multiple features are reweighted stepwise so that the resulting state becomes increasingly aligned with the intended B or D endpoint.

This means the project currently supports:
- endpoint-directed movement
- substantial A-structure retention
- distributed coefficient change across several features
- stepwise accumulation, not just final-query change
- a main inside-A mechanism with smaller but selective outside-A support

Current Q1-focused extension:
- in the wider top30 inside-A view, the useful mechanistic unit may be a co-moving module rather than an individual PCA axis
- the strongest cautious reading is therefore:
  - context may reallocate priority across a small number of latent inside-A modules
  - those modules can include direct endpoint-push, shared support, and suppression-like roles

## What This Claim Rejects

The current analysis pushes against two overly simple readings.

### 1. One-Axis Replacement

Rejected form:
- A is replaced by a single B axis or D axis

Why this is too weak:
- residual structure remains
- change spreads across several feature coordinates
- inside/outside decomposition shows mixed structure rather than pure collapse

### 2. Outside-A As Pure Noise

Rejected form:
- only inside-A matters, outside-A is meaningless leftover noise

Why this is too weak:
- outside-A often remains endpoint-selective
- local tangent checks suggest it is not just a global linear-basis artifact
- strict same-state intervention framing treats it as a smaller but structured support component

## Core Conceptual Objects

### A-only basis

The key theoretical move is to build a basis from A-only states first.

Notation:
- `G_A`: basis for the original A-only state family

Interpretation:
- this basis describes what A already is before mixed context changes it

### Update directions

Separate from the state basis are directions that describe how context changes A.

Interpretation:
- state basis answers "what A contains"
- update directions answer "how context pushes A"

These are not competing explanations.

## Core Reweighting View

The intended model is:
- A-only states live in an A-structured feature space
- mixed states still retain significant projection into that space
- context changes the coefficients on that basis rather than deleting the basis entirely

In practical terms:
- some A features are amplified
- some remain relatively stable
- some decrease
- an additional component may appear outside the A-space

The mixed state becomes more B-like or D-like because the relevant bundle is strengthened, not because all non-target structure disappears.

## Why Stepwise Analysis Matters

A final-query-only view is not enough.

The stronger claim is:
- the reweighting begins during prompt progression
- the intended endpoint dominance accumulates across A positions
- the mechanism is therefore process-level, not only a final-state artifact

This is why the project extracts matched A states across:
- `A_demo_1`
- `A_demo_2`
- `A_demo_3`
- `A_demo_4`
- `A_query`

## Strongest Supported Interpretation By Layer

### Layer 1. Endpoint Movement

Supported:
- mixed states move in the intended direction

Not supported:
- a single clean line explains everything

### Layer 2. Query-Only Reweighting

Supported:
- final mixed states still retain A-structure
- coefficient change is distributed across multiple features

### Layer 3. Stepwise Reweighting

Supported:
- reweighting accumulates across prompt positions

### Layer 4. Endpoint-Aligned Contribution

Supported:
- coefficient changes are not only large; they are increasingly aligned with intended endpoint directions

### Layer 5. Inside / Outside Split

Supported:
- inside-A carries most of the bulk
- outside-A remains smaller but selective

### Layer 6. Local Tangent / Curvature Checks

Supported:
- outside-A does not disappear under local-basis variants

## Present Best Reading

The safest current summary is:

> Inside-A structured reweighting is the main mechanism.
> Outside-A is not the main carrier, but it is also not well described as meaningless noise.
> It behaves more like a smaller selective support component that sharpens, disambiguates, or complements the endpoint shift.

For Q1 specifically, a stronger but still cautious extension is:

> The feature-level picture can be compressed into a small number of co-moving inside-A modules.
> In that reading, context is not only changing individual coefficients; it is reallocating module priority across direct endpoint-push, shared-support, and suppression-like groups.

## Scope Boundary

This theory page is about representation change, not about every downstream behavioral claim.

It does not by itself prove:
- broad portability
- all-layer causal mediation in the native model
- every behavioral phenomenon in PT space

It is a theory of the representation-analysis branch built on condition-qwise artifacts.
